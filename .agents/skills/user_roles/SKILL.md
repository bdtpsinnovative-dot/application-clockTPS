# NexHR User Roles and Permission System

เอกสารนี้ระบุระดับสิทธิ์การใช้งาน (User Roles) ภายในระบบ NexHR ซึ่งแบ่งออกเป็น 3 บทบาทหลัก เพื่อให้ AI และนักพัฒนาออกแบบระบบควบคุมสิทธิ์การเข้าถึง (Access Control) ได้อย่างถูกต้อง

---

## 1. บทบาทของผู้ใช้งาน (User Roles)

ในระบบจะถูกกำหนดไว้ในตารางฐานข้อมูล `users` (ฟิลด์ `role`) ซึ่งมี 3 กลุ่มดังนี้:

### 👥 1. พนักงานทั่วไป (Employee)
* **เป้าหมาย:** สมาชิกทั่วไปขององค์กรที่ต้องการบันทึกเวลาทำงานและขออนุมัติส่วนตัว
* **สิทธิ์การใช้งาน:**
  * ลงเวลาเข้า-ออกงาน (Clock-In / Clock-Out) ผ่านระบบ Geofencing และแสกนใบหน้า
  * ส่งคำขออนุมัติลา (Leave Requests) และคำขอทำงานนอกสถานที่ (Offsite Requests) ของตนเอง
  * ดูประวัติการลงเวลาทำงานและปฏิทินงาน/วันหยุดส่วนตัว
  * ดูโปรไฟล์และแก้ไขข้อมูลทั่วไปของตัวเอง

### 💼 2. ฝ่ายทรัพยากรบุคคล (HR)
* **เป้าหมาย:** ผู้ดูแลจัดการงานด้านบุคคล รายงานการลงเวลา และการอนุมัติใบลา
* **สิทธิ์การใช้งาน:**
  * สิทธิ์เหมือน **Employee** ทุกประการ (สามารถลงเวลาทำงานของตัวเองได้)
  * ดูรายงานการเข้างานของพนักงานทั้งหมดในแผนก/องค์กร
  * อนุมัติหรือปฏิเสธคำขอลา (Leave Requests) และคำขอทำงานนอกสถานที่ (Offsite Requests)
  * จัดการข้อมูลโควตาวันลา (Leave Quotas) ของพนักงาน

### 🛡️ 3. ผู้ดูแลระบบ (Admin)
* **เป้าหมาย:** ผู้ควบคุมระบบระดับสูงสุด จัดการโครงสร้างและนโยบายองค์กร
* **สิทธิ์การใช้งาน:**
  * สิทธิ์เหมือน **HR** และ **Employee** ทุกประการ
  * จัดการผู้ใช้งานในระบบ (เพิ่ม/ลด/ระงับบัญชีผู้ใช้ หรือแก้ไขสิทธิ์การเข้าถึง)
  * จัดการสถานที่ทำงานและพิกัด Geofencing (`work_locations`)
  * มอบหมายงาน (Tasks) ให้กับพนักงาน และติดตามความคืบหน้างานบนบอร์ด
  * จัดการการตั้งค่าความปลอดภัยและเทมเพลตระบบ

---

## 🔒 แนวทางการควบคุมการเข้าถึงในโค้ด (Access Control Guidelines)

* **ระดับ Database:** 
  * ตรวจสอบบทบาทของสิทธิ์ `role` ทุกครั้งก่อนดึงข้อมูลรายงานรวมหรือตารางสำหรับ Admin/HR
* **ระดับ UI (Flutter):**
  * แสดงแท็บเมนูจัดการระบบเฉพาะผู้ใช้ที่มีบทบาทเป็น `admin` หรือ `hr` เท่านั้น
  * หน้าจัดการพิกัด Geofencing และหน้ามอบหมายงาน `AdminTasksPage` สงวนสิทธิ์ไว้ให้เฉพาะ `admin` เท่านั้น
