# NexHR UI Design Guidelines

คู่มือการออกแบบส่วนติดต่อผู้ใช้ (UI/UX) ของแอปพลิเคชัน NexHR เพื่อให้หน้าจอทั้งหมดมีมิติ ความสวยงาม และอยู่ในโทนสีเดียวกัน (Theme Consistency)

---

## 🎨 ระบบสี (Color System)

เราใช้จานสี (Color Palette) หลักดังต่อไปนี้:

| สี | Hex Code | ชื่อตัวแปร (Flutter) | การใช้งาน |
|---|---|---|---|
| **Deep Blue** | `#2563EB` | `workBlue` | สีหลักของระบบ (Primary Accent), ปุ่มกดหลัก, จุดเน้นสำคัญ |
| **Sky Blue** | `#0EA5E9` | `workSky` | ไล่เฉดสีคู่กับ Deep Blue, แท็บสีแสดงพื้นที่ทำงาน |
| **Background**| `#F8FAFC` | `workBackground` | สีพื้นหลังหลักของหน้าจอ |
| **Dark Slate** | `#1E293B` | `workText` | สีของตัวหนังสือหลัก (Primary Text) |
| **Muted Slate**| `#94A3B8` | `workMuted` | สีของตัวหนังสือคำอธิบายหรือไอคอนสถานะทั่วไป |

## 📦 รูปแบบการ์ดและเลย์เอาต์ (Cards & Layouts)

### 1. โครงสร้างการ์ด (WorkCard)
ทุกหน้าจอควรบรรจุข้อมูลสำคัญในการ์ดที่มีดีไซน์สวยงามและเรียบง่าย โดยมีโครงสร้างดังนี้:
* **มุมโค้ง (Border Radius)**: `24`
* **สีกรอบ (Border Color)**: `#F1F5F9` (บางเฉียบกลมกลืน)
* **เงา (Box Shadow)**:
  * สีเงา: `#0F172A` (ความโปร่งใส 5% หรือ `Color(0x0D0F172A)`)
  * ความฟุ้ง (Blur Radius): `10`
  * ระยะเยื้อง (Offset): `(0, 3)`

### 2. หัวข้อด้านบน (WorkHeader)
ส่วนหัวของทุกเพจควรเป็นแบบ Gradient จากบนซ้ายลงล่างขวา (สี `workBlue` ไล่ไป `workSky`) เพื่อให้ผู้ใช้สัมผัสถึงความน่าใช้งานและเป็นโทนเดียวกัน

---

## 🔄 โครงสร้างการนำทางหลัก (Main Navigation)

เมื่อผู้ใช้เข้าสู่ระบบสำเร็จ จะพบกับแถบนำทาง (Bottom Navigation / NavigationBar) ที่มีทั้งหมด **6 เมนูย่อย**:

1. **หน้าหลัก (Home)**:
   * **สำหรับผู้ใช้ทั่วไป**: หน้าแสดงการต้อนรับ ข้อมูลสรุปของตัวเอง และข่าวกิจกรรม
   * **สำหรับผู้ดูแล (Admin/HR)**: หน้าแสดงสถิติการเข้างานพนักงานวันนี้ จำนวนผู้รอการอนุมัติ และปุ่มเมนูลัดสำหรับแอดมิน
2. **ลงเวลา (Clock-In)**: หน้าจอลงเวลาเข้างาน/ออกงาน ตรวจสอบพื้นที่ Geofencing และแสกนใบหน้า
3. **คำขอ (Requests)**: หน้าส่งใบลา/ออกนอกสถานที่ และดูประวัติคำขอของตัวเอง
4. **ปฏิทิน (Calendar)**: ดูตารางงาน วันหยุดนักขัตฤกษ์ และประวัติการลาบนหน้าปฏิทิน
5. **แจ้งเตือน (Notifications)**: หน้าแสดงรายการแจ้งเตือน เช่น คำขอได้รับการอนุมัติ, แจ้งเตือนเวลาเข้างาน
6. **โปรไฟล์ (Profile)**: ข้อมูลส่วนตัวและการออกจากระบบ

---

## 📐 ข้อตกลงการเขียนโค้ดสำหรับ UI
* หลีกเลี่ยงการใช้สีแบรนด์อื่นๆ ที่ไม่เกี่ยวข้อง
* การสร้างส่วนต้อนรับพนักงานควรใช้การทักทายที่เป็นมิตร เช่น *"สวัสดีวันศุกร์ครับ คุณศิริวิชญ์"*
* ตัวปุ่มกดและฟีลด์กรอกข้อมูลจะต้องมีความสูงเฉลี่ย `48 - 56 px` และมีมุมโค้งที่สอดคล้องกัน (`12 - 14 px`)
